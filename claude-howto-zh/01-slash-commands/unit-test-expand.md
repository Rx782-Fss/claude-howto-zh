# 扩展单元测试

根据项目的测试框架扩展现有的单元测试:

1. **分析覆盖率**:运行覆盖率报告,识别未测试的分支、边界情况和低覆盖率区域
2. **识别缺口**:审查代码中的逻辑分支、错误路径、边界条件、空值/空输入
3. **编写测试**,使用项目自身的测试框架:
   - Jest/Vitest/Mocha(JavaScript/TypeScript)
   - pytest/unittest(Python)
   - Go testing/testify(Go)
   - Rust test framework(Rust)
4. **针对特定场景**:
   - 错误处理和异常
   - 边界值(最大/最小值、空值、null)
   - 边角情况和极端情况
   - 状态转换和副作用
5. **验证改进效果**:再次运行覆盖率,确认有可量化的提升

仅输出新测试代码块。遵循已有的测试模式和命名规范。

> 💡 **中文开发者提示**:测试覆盖率不是越高越好,应聚焦于核心业务逻辑和关键路径。建议目标覆盖率:核心模块 ≥80%,整体 ≥60%。使用 `--cov-report=term-missing` 可以直观看到哪些行未被覆盖。
