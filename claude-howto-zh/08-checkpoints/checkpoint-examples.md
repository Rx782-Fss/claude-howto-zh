# Checkpoint 使用示例

在 Claude Code 中有效使用检查点的真实案例。

注意:每次用户输入时检查点会自动创建。你无需手动保存。要回退,按两次 `Esc` 键(`Esc+Esc`)或使用 `/rewind` 打开检查点浏览器。

## 示例 1:数据库迁移

### 场景
你需要从 MySQL 迁移到 PostgreSQL,但想同时探索直接迁移和渐进式双写两种方案。

### 工作流程

```
用户: 让我们实现从 MySQL 到 PostgreSQL 的直接迁移
     [自动创建了检查点 A]

Claude: 我来实现直接迁移...
[修改了数据库驱动、更新了查询语句、调整了 Schema]

用户: 运行测试
     [自动创建了检查点 B]

Claude: 正在运行测试...
❌ 15 个测试失败 -- 连接池问题

用户: 这种方案有问题。我们改用双写模式吧。
[用户按下 Esc+Esc,选择检查点 A,选择"恢复代码和对话"]

用户: 现在实现双写模式,同时向两个数据库写入数据

Claude: 我来实现双写迁移...
[实现了双写层、渐进式迁移策略]

用户: 运行测试

Claude: 正在运行测试...
✅ 所有测试通过

用户: 双写方案明显更安全。我们来提交吧。
```

## 示例 2:性能优化

### 场景
尝试多种性能优化策略。

### 工作流程

```
用户: 这个 API 接口太慢了,需要优化
     [自动创建了检查点 A]

Claude: 让我分析一下性能瓶颈...
[添加了 Redis 缓存层]
     [自动创建了检查点 B]

用户: 测试一下效果
Claude: 响应时间从 500ms 降到 150ms ✅

用户: 还能更快吗?试试内存缓存
[用户回退到检查点 A]

Claude: 改用内存缓存方案...
[实现 LRU 内存缓存]
     [自动创建了检查点 C]

用户: 测试一下
Claude: 响应时间从 500ms 降到 80ms 🚀

用户: 太好了!用这个方案。
```
