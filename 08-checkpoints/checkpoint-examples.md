# Checkpoint 实战示例

在 Claude Code 中有效使用 Checkpoints 的真实场景示例。

注意：Checkpoints 在每次用户输入时自动创建，无需手动保存。要回退，按两次 `Esc`（`Esc+Esc`）或使用 `/rewind` 打开检查点浏览器。

## 示例 ①：数据库迁移

### 场景
你需要从 MySQL 迁移到 PostgreSQL，但想探索直接迁移和渐进式双写两种方案。

### 工作流

```
用户: 我们来实现从 MySQL 到 PostgreSQL 的直接迁移吧
     [检查点 A 自动创建]

Claude: 我来实现直接迁移...
[修改数据库驱动、更新查询语句、调整 schema]

用户: 运行测试
     [检查点 B 自动创建]

Claude: 运行测试中...
❌ 15 个测试失败 —— 连接池问题

用户: 这个方案有问题。我们试试双写模式吧。
[用户按 Esc+Esc，选择检查点 A，选择 "恢复代码和对话"]

用户: 现在实现同时写入两个数据库的双写模式

Claude: 我来实现双写迁移...
[实现双写层、渐进式迁移策略]

用户: 运行测试

Claude: 运行测试中...
✅ 所有测试通过

用户: 双写方案明显更安全。我们提交这个方案吧。
```

## 示例 ⑧：使用"从此处总结"

### 场景
经过长时间调试后，你想压缩对话内容同时保留上下文。

### 工作流

```
用户: [经过 20 多条消息的调试和探索后]

[用户按 Esc+Esc，选择早期检查点，选择 "从此处总结"]
[可选地提供指令："聚焦于我们尝试过什么以及什么有效"]

Claude: [生成从该点开始的对话摘要]
[原始消息保留在记录中]
[摘要替换可见的对话，减少上下文窗口占用]

用户: 现在我们继续用有效的那个方案。
```

## 关键要点

① **Checkpoints 是自动的**——每次用户输入都会创建检查点，无需手动保存
② **使用 Esc+Esc 或 /rewind**——这是访问检查点浏览器的两种方式
③ **选择正确的恢复选项**——根据需要选择恢复代码、对话、两者都恢复或总结
④ **不要害怕实验**——Checkpoints 让你可以安全地尝试激进变更
⑤ **与 git 结合使用**——用 Checkpoints 进行探索，用 git 保存最终成果
⑥ **总结长会话**——使用"从此处总结"保持对话的可管理性
