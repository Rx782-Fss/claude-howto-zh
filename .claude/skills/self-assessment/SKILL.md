# 自我评估与学习路径顾问

全面的交互式评估工具，跨 10 个功能领域评估 Claude Code 熟练度，识别具体的技能差距，并生成个性化的学习路径来提升等级。

## 指令

### 步骤 1：欢迎与选择评估模式

向用户展示评估深度选择：

使用 AskUserQuestion 提供以下选项：
- **快速评估** — "8 道题，约 2 分钟。确定你的整体级别（入门/中级/高级）并提供学习路径。"
- **深度评估** — "5 个类别含详细问题，约 5 分钟。提供各主题的技能评分、识别具体差距、构建优先级学习路径。"

如果用户选择 **快速评估**，转到步骤 2A。
如果用户选择 **深度评估**，转到步骤 2B。

---

### 步骤 2A：快速评估

展示两道多选题（AskUserQuestion 每题最多支持 4 个选项）：

**问题 1**（标题："基础"）：
"第 1/2 部分：你已掌握以下哪些 Claude Code 技能？"
选项：
1. "启动 Claude Code 并对话" — 我可以运行 `claude` 并与之交互
2. "创建/编辑过 CLAUDE.md" — 我设置过项目或个人记忆
3. "使用过 3+ 个斜杠命令" — 例如 /help、/compact、/model、/clear
4. "创建过自定义命令/技能" — 编写过 SKILL.md 或自定义命令文件

**问题 2**（标题："进阶"）：
"第 2/2 部分：你已掌握以下哪些进阶技能？"
选项：
1. "配置过 MCP 服务器" — 例如 GitHub、数据库或其他外部数据源
2. "设置过钩子" — 在 ~/.claude/settings.json 中配置过钩子
3. "创建/使用过子代理" — 使用 .claude/agents/ 进行任务委派
4. "使用过打印模式（claude -p）" — 使用 `claude -p` 进行非交互式或 CI/CD 用途

**计分规则：**
- 总计 0-2 分 = 级别 1：入门
- 总计 3-5 分 = 级别 2：中级
- 总计 6-8 分 = 级别 3：高级

带着级别结果进入步骤 3，列出哪些具体项目未被勾选为差距。

---

### 步骤 2B：深度评估

进行 5 轮问题提问，每轮调用一次 AskUserQuestion。每轮涵盖 2 个相关功能领域。所有轮次均使用多选。

**重要提示**：AskUserQuestion 每个问题最多支持 4 个选项。每轮恰好有 1 个问题，包含 4 个选项，覆盖 2 个主题（每个主题 2 个选项）。

---

**第 1 轮 — 斜杠命令 & 记忆**（标题："Commands"）

"你做过以下哪些操作？请选择所有适用的项。"
选项：
1. "创建过自定义斜杠命令或技能" — 编写过带 frontmatter 的 SKILL.md 文件，或创建过 .claude/commands/ 文件
2. "在命令中使用过动态上下文" — 在技能/命令文件中使用过 `$ARGUMENTS`、`$0`/`$1`、反引号 `!command` 语法或 `@file` 引用
3. "设置过项目 + 个人记忆" — 同时创建了项目 CLAUDE.md 和个人 ~/.claude/CLAUDE.md（或 CLAUDE.local.md）
4. "使用过记忆层级特性" — 了解 7 级优先顺序，使用过 .claude/rules/ 目录、路径特定规则或 @import 语法

**第 1 轮计分：**
- 选项 1-2 对应 **斜杠命令**（0-2 分）
- 选项 3-4 对应 **记忆**（0-2 分）

---

**第 2 轮 — 技能 & 钩子**（标题："Automation"）

"你做过以下哪些操作？请选择所有适用的项。"
选项：
1. "安装并使用过自动触发的技能" — 根据其描述字段自动触发而无需手动 /command 调用的技能
2. "控制过技能调用行为" — 在 SKILL.md frontmatter 中使用过 `disable-model-invocation`、`user-invocable` 或 `context: fork` 配合 agent 字段
3. "设置过 PreToolUse 或 PostToolUse 钩子" — 配置过在工具执行前后运行的钩子（如命令验证器、自动格式化器）
4. "使用过高级钩子特性" — 配置过基于提示词的钩子、SKILL.md 中的组件作用域钩子、HTTP 钩子或带自定义 JSON 输出的钩子（updatedInput、systemMessage）

**第 2 轮计分：**
- 选项 1-2 对应 **技能**（0-2 分）
- 选项 3-4 对应 **钩子**（0-2 分）

---

**第 3 轮 — MCP & 子代理**（标题："Integration"）

"你做过以下哪些操作？请选择所有适用的项。"
选项：
1. "连接过 MCP 服务器并使用其工具" — 例如用于 PR/Issue 的 GitHub MCP、用于查询的数据库 MCP 或任何外部数据源
2. "使用过高级 MCP 特性" — 项目级 .mcp.json、OAuth 认证、带 @mentions 的 MCP 资源、Tool Search 或 `claude mcp serve`
3. "创建或配置过自定义子代理" — 在 .claude/agents/ 中定义了带有自定义工具、模型或权限的代理
4. "使用过高级子代理特性" — Worktree 隔离、持久化代理内存、后台任务（Ctrl+B）、带 `Task(agent_name)` 的代理白名单或代理团队

**第 3 轮计分：**
- 选项 1-2 对应 **MCP**（0-2 分）
- 选项 3-4 对应 **子代理**（0-2 分）

---

**第 4 轮 — 检查点 & 高级功能**（标题："Power User"）

"你做过以下哪些操作？请选择所有适用的项。"
选项：
1. "使用检查点进行安全实验" — 创建过检查点，使用过 Esc+Esc 或 /rewind，恢复过代码和/或对话，或使用过 Summarize 选项
2. "使用过规划模式或扩展思考" — 通过 /plan、Shift+Tab 或 --permission-mode plan 激活规划；通过 Alt+T/Option+T 切换扩展思考
3. "配置过权限模式" — 通过 CLI 标志、键盘快捷键或设置使用过 acceptEdits、plan、dontAsk 或 bypassPermissions 模式
4. "使用过远程/桌面/Web 功能" — 使用过 `claude remote-control`、`claude --remote`、`/teleport`、`/desktop` 或配合 `claude -w` 使用 worktree

**第 4 轮计分：**
- 选项 1 对应 **检查点**（0-1 分）
- 选项 2-4 对应 **高级功能**（0-3 分，上限 2 分）

---

**第 5 轮 — 插件 & CLI**（标题："Mastery"）

"你做过以下哪些操作？请选择所有适用的项。"
选项：
1. "安装或创建过插件" — 从市场使用过捆绑插件，或创建了带 plugin.json 清单的 .claude-plugin/ 目录
2. "使用过插件高级特性" — 插件钩子、插件 MCP 服务器、LSP 配置、插件命名空间命令或用于测试的 --plugin-dir 标志
3. "在脚本或 CI/CD 中使用过打印模式" — 使用过 `claude -p` 配合 --output-format json、--max-turns、管道输入或集成到 GitHub Actions / CI 流水线中
4. "使用过高级 CLI 特性" — 会话恢复（-c/-r）、--agents 标志、--json-schema 结构化输出、--fallback-model、--from-pr 或批处理循环

**第 5 轮计分：**
- 选项 1-2 对应 **插件**（0-2 分）
- 选项 3-4 对应 **CLI**（0-2 分）

---

### 步骤 3：计算并呈现结果

#### 3A：针对快速评估

统计总勾选数并确定级别。然后呈现：

```markdown
## Claude Code 技能评估结果

### 你的级别：[Level 1: 入门 / Level 2: 中级 / Level 3: 高级]

你勾选了 **N/8** 项。

[基于级别的单行激励性总结]

### 你的技能概况

| 领域 | 状态 |
|------|------|
| 基础 CLI 与对话 | [已掌握/差距] |
| CLAUDE.md 与记忆 | [已掌握/差距] |
| 斜杠命令（内置）| [已掌握/差距] |
| 自定义命令与技能 | [已掌握/差距] |
| MCP 服务器 | [已掌握/差距] |
| 钩子 | [已掌握/差距] |
| 子代理 | [已掌握/差距] |
| 打印模式与 CI/CD | [已掌握/差距] |

### 已识别的差距

[对于每项未勾选的项目，提供一行描述需要学习的内容及教程链接]

### 你的个性化学习路径

[输出特定于该级别的学习路径 —— 参见步骤 4]
```

#### 3B：针对深度评估

从 5 轮中计算各主题分数。每个主题得 0-2 分。然后呈现：

```markdown
## Claude Code 技能评估结果

### 整体级别：[Level 1 / Level 2 / Level 3]

**总分：N/20 分**

[单行激励性总结]

### 你的技能概况

| 功能领域 | 分数 | 掌握度 | 状态 |
|---------|------|--------|------|
| 斜杠命令 | N/2 | [无/基础/熟练] | [学习/复习/已精通] |
| 记忆 | N/2 | [无/基础/熟练] | [学习/复习/已精通] |
| 技能 | N/2 | [无/基础/熟练] | [学习/复习/已精通] |
| 钩子 | N/2 | [无/基础/熟练] | [学习/复习/已精通] |
| MCP | N/2 | [无/基础/熟练] | [学习/复习/已精通] |
| 子代理 | N/2 | [无/基础/熟练] | [学习/复习/已精通] |
| 检查点 | N/1 | [无/熟练] | [学习/已精通] |

**掌握度说明**：0 = 无，1 = 基础，2 = 熟练

### 强项领域
[列出得分为 2/2 的主题 —— 这些已精通]

### 优先差距（下一步学习）
[列出得分为 0 的主题 —— 这些需优先关注，按依赖关系排序]
```

**深度评估整体级别计算：**
- 总分 0-6 = Level 1：入门
- 总分 7-13 = Level 2：中级
- 总分 14-20 = Level 3：高级

### 步骤 4：生成个性化学习路径

根据评估结果生成专门适应用户差距的学习路径。不要只是重复通用级别路径 —— 要做适配。

#### 路径生成规则

1. **跳过已精通主题**：如果某主题得分 2/2，不将其纳入路径。
2. **按依赖顺序排列优先级**：斜杠命令在技能之前，记忆在子代理之前等。依赖顺序为：
   - 斜杠命令（无依赖）→ 技能（依赖于斜杠命令）
   - 记忆（无依赖）→ 子代理（依赖于记忆）
   - CLI 基础（无依赖）→ CLI 精通（依赖于全部）
   - 检查点（无依赖）
   - 钩子（依赖于斜杠命令）
   - MCP（无依赖）→ 插件（依赖于 MCP、技能、钩子）
   - 高级功能（依赖于前面所有内容）
3. **对于得分 1/2 的主题**：建议"深入钻研"——链接到他们缺失的具体高级部分。
4. **估算时间**：仅累加他们需要学习/复习的主题所需时间。
5. **分组为阶段**：将剩余主题组织为每组 2-3 个主题的逻辑阶段。

#### 路径输出格式

```markdown
### 你的个性化学习路径

**预估时间**：~N 小时（根据你当前技能调整）

#### 第 1 阶段：[阶段名称] (~N 小时)
[仅当这些领域存在差距时]

**[主题名称]** — [从头学起 / 深入钻研高级功能]
- 教程：[教程目录链接]
- 重点：[他们需要的具体章节/概念]
- 关键练习：[一个具体的练习任务]
- 完成标志：[具体的成功标准]

**[主题名称]** — ...

---

#### 第 2 阶段：[阶段名称] (~N 小时)
...
---

### 推荐实践项目

根据你的差距，尝试以下真实练习来巩固所学：

1. **[项目名称]**：[结合 2-3 个差距主题的一行描述]
2. **[项目名称]**：[一行描述]
3. **[项目名称]**：[一行描述]
```

---

### 步骤 5：提供后续行动

呈现结果后，询问用户接下来想做什么：

使用 AskUserQuestion 提供以下选项：
- **开始学习** — "帮助我立即开始学习路径中的第一个主题"
- **深入钻研某个差距** — "详细解释我的一个差距领域以便我在此学习"
- **实践项目** — "搭建一个覆盖我差距领域的练习项目"
- **重新测评** — "我想重新测验（也许换另一种模式）"

如果选择 **开始学习**：阅读第一个差距教程的 README.md 并引导用户完成第一个练习。
如果选择 **深入钻研某个差距**：询问哪个差距主题，然后阅读相关教程 README.md 并用示例解释关键概念。
如果选择 **实践项目**：设计一个结合 2-3 个差距主题的小型项目并提供具体步骤。
如果选择 **重新测评**：返回步骤 1。

## 错误处理

### 用户在某轮未选择任何项
将该轮主题视为 0 分。继续下一轮。

### 用户在所有轮次中都未选择任何项
分配 Level 1：入门级别。鼓励从头开始。输出完整的 Level 1 学习路径。

### 用户想重新测评
从步骤 1 重新开始全新评估。

### 用户不同意自己的级别
认可他们的偏好。询问他们认同哪个级别。为他们选择的级别呈现学习路径，并对他们可能遗漏的主题进行前置条件检查。

### 用户询问特定主题
如果用户在评估过程中说类似"告诉我关于钩子的事"或"我想学 MCP"，记录下来。呈现结果后，无论分数如何都在他们的学习路径中突出显示该主题。
