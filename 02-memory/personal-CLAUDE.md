# 我的开发偏好设置

## 关于我
- **经验水平**:8 年全栈开发经验
- **偏好语言**:TypeScript、Python
- **沟通风格**:直接明了,配合示例说明
- **学习风格**:偏好可视化图表 + 代码结合的方式

## 代码偏好

### 错误处理
我偏好使用 try-catch 进行显式错误处理,并给出有意义的错误信息。
避免使用通用错误。始终记录错误日志以便调试。

### 注释规范
注释用于解释"为什么",而不是"做什么"。代码本身应该是自解释的。
注释应该说明业务逻辑或非显而易见的决策原因。

### 测试方式
我偏好 TDD(测试驱动开发)。
先写测试,再写实现。
关注行为测试,而非实现细节。

### 架构设计
我偏好模块化、松耦合的设计。
使用依赖注入以提高可测试性。
分离关注点(控制器 Controller、服务 Service、仓库 Repository)。

## 调试偏好
- 使用 console.log 并加上 `[DEBUG]` 前缀
- 包含上下文信息:函数名、相关变量
- 在可用时使用堆栈跟踪
- 日志中始终包含时间戳

## 沟通方式
- 用图表解释复杂概念
- 先展示具体示例,再讲解理论
- 包含修改前后的代码对比
- 在结尾总结关键要点
