# 代码优化分析

对当前项目进行全面的代码质量分析:

1. **分析复杂度**:识别高圈复杂度区域、深层嵌套和长函数
2. **检查性能瓶颈**:查找 N+1 查询、不必要的计算、低效算法
3. **审查可读性**:检查命名规范、函数长度、注释质量
4. **发现重复代码**:找出可以提取为共享工具的重复模式
5. **提出改进建议**:提供具体的重构建议,附带前后对比

**输出格式**:
```
## 📊 优化分析报告

### 🔴 高优先级
- [文件]:[行号] - 问题说明 → 建议方案

### 🟡 中优先级
...

### 🟢 低优先级
...

### ✅ 已做得好的部分
...
```

> 💡 **中文开发者提示**:此命令非常适合在代码审查前运行,或在重构前了解代码库现状。它会帮你快速定位需要优化的热点区域。
